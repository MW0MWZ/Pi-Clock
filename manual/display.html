Display Layers

All 15 layers that make up the Pi-Clock world map display.

The Pi-Clock display is built from independent layers, composited from back to front. Each layer can be toggled on or off and (for most layers) have its opacity adjusted from 0–100%. Layers are only re-rendered when their data changes, so static layers like grids and borders have zero ongoing CPU cost.

Layers are configured through the web dashboard or by editing /data/etc/pi-clock-layers.conf directly. Changes take effect immediately — no reboot required.

Enabling layers on a Pi Zero W: All layers work on every Pi model, but enabling many layers simultaneously on a single-core Pi Zero W will reduce the frame rate. Start with the defaults and add layers one at a time. See the Hardware Guide for performance details.

Core Map Layers

Base Map (Night)

Static Always On No Network

The base layer renders the NASA Black Marble (city lights at night) image as a full-screen equirectangular map. This forms the background that all other layers are composited on top of.

The base map supports the center longitude setting — the map wraps horizontally so you can center it on your part of the world (e.g., set to -90 to center on the Americas, or +140 for the Pacific).

Data source: NASA Black Marble (Earth at Night), provided by the pi-clock-maps package at 720p, 1080p, 1440p, and 4K resolutions.

Daylight (Day/Night Blend)

Updates: 60s Always On No Network

This layer composites the NASA Blue Marble (daytime satellite imagery) over the sunlit hemisphere, blending smoothly across the twilight zone to create the greyline terminator effect. The result is a photorealistic day/night view of the Earth that updates every 60 seconds as the terminator moves.

The solar position is calculated using the Meeus algorithm (accurate to within a few arcseconds) and the illumination is computed per-pixel for a smooth gradient from day to night.

Data source: NASA Blue Marble Next Generation (daytime). Solar math is computed offline — no network needed.

Performance note: At higher resolutions, the daylight calculation uses adaptive sampling (4-pixel steps at 1080p, 8-pixel at 4K) to reduce the number of trigonometric calculations per frame.

Country Borders

Static Default: On No Network

Draws country boundary lines over the map. Loaded from a binary borders data file derived from Natural Earth 10m cultural vectors.

Dashboard: Toggle on/off, adjust opacity.

Grid & Zone Overlays

Lat/Lon Grid (Graticule)

Static Default: Off No Network

A latitude/longitude reference grid drawn every 10 degrees. Special lines are emphasised: the Equator, Prime Meridian, Tropics of Cancer and Capricorn, and the Arctic and Antarctic circles. Grid intersection points are labelled with their coordinates.

Dashboard: Toggle on/off, adjust opacity.

Time Zones

Static Default: On No Network

Real-world time zone boundaries following political borders, drawn as dotted blue lines. Each zone is labelled with its UTC offset (UTC-12 through UTC+14). Based on the Natural Earth 10m time zones dataset.

Dashboard: Toggle on/off, adjust opacity.

CQ Zones

Static Default: Off No Network

The 40 CQ zones used in DX contests (CQ WW, CQ WPX, etc.). Drawn as dashed orange boundary lines with zone numbers positioned at each zone's centroid. Useful for contest operators who need to quickly identify which CQ zone a station is in.

Dashboard: Toggle on/off, adjust opacity.

Maidenhead Grid

Static Default: Off No Network

The Maidenhead locator system grid used universally in amateur radio for location exchange. Displays the 18×18 field grid (each field is 20° longitude × 10° latitude) with two-letter field labels (AA through RR). Drawn as dashed green lines.

Dashboard: Toggle on/off, adjust opacity.

Celestial Body Markers

Sun Position

Updates: 60s Default: Off No Network

Marks the sub-solar point (where the sun is directly overhead) with a filled yellow/orange circle and eight radiating rays. The marker moves approximately 15° westward per hour and varies north/south with the seasons. Marker size scales with the display resolution.

Dashboard: Toggle on/off, adjust opacity.

Moon Position & Phase

Updates: 60s Default: Off No Network

Marks the sub-lunar point with a moon icon that shows the current phase. The illuminated portion is drawn as a partial fill (crescent, quarter, gibbous, or full), accurately reflecting the real-time moon phase. The dark (unlit) portion is shown in grey.

Dashboard: Toggle on/off, adjust opacity.

Ham Radio Data Layers

DX Cluster Spots

Updates: 30–60s Default: Off Requires Network

Connects to a DX cluster server via telnet and plots live DX spots on the map. Each spot is shown as a band-coloured dot at the DX station's location, with an optional great-circle arc from the spotter to the DX station. Callsign labels are displayed with dark background pills for readability.

Spot Ageing

Spots fade in opacity as they approach their expiry time. In the final 5 minutes before a spot expires, it gradually becomes more transparent. The maximum spot age is configurable (default: 15 minutes).

Configuration

  • Cluster server — hostname and port (default: dxc.ve7cc.net:7373)
  • Distance filter — maximum distance from your QTH in km (0 = no filter)
  • Band filter — enable/disable individual bands (160m through 2m)
  • Spot age — how long spots remain on the map (5, 10, 15, 30, or 60 minutes)

Dashboard: Toggle on/off, adjust opacity. DX settings card appears when this layer is enabled. See DX Cluster Settings for full configuration details.

Data source: Any DX Spider or AR Cluster server via telnet. Callsign-to-location resolution uses the cty.dat prefix database (AD1C/ARRL).

Propagation Heat Map

Updates: 10s (band cycle) Default: Off Uses NOAA SFI

Displays an HF band propagation heat map across the entire map surface. The map shows signal reliability from your QTH to every 2-degree grid cell, using a simplified VOACAP ionospheric model that incorporates live solar flux data.

How It Works

The heat map automatically cycles through the amateur HF bands (80m through 10m) every 10 seconds, showing which parts of the world are reachable on each band at the current time. The model considers:

  • F2 layer critical frequency (based on latitude and sunspot number)
  • D-layer absorption (time-of-day and latitude dependent)
  • Multi-hop propagation (obliquity factors for different distances)
  • Geomagnetic storm degradation (using live Kp index)

Reading the Heat Map

The heat map uses a single colour with varying opacity to show propagation likelihood. Areas with higher opacity (more solid colour) indicate a higher probability of a usable signal path. Areas with lower opacity (more transparent, showing the map underneath) indicate propagation is less likely. Completely transparent areas have no predicted propagation on that band at the current time.

Configuration

  • Band selection — enable/disable individual bands in the cycle (80m through 10m)
  • Antenna gain — your antenna gain in dBi (affects predicted reliability)

Dashboard: Toggle on/off, adjust opacity. Propagation settings card appears when enabled.

Offline operation: The propagation model works offline using default solar values (SFI=100). When connected to the internet, it uses live SFI and Kp data from NOAA for more accurate predictions.

Satellite Tracking

Updates: Real-time Default: Off Requires Network (for TLEs)

Tracks amateur radio satellites in real-time using SGP4 orbit propagation. Each enabled satellite is shown with:

  • A filled dot at the current sub-satellite point
  • A ground track showing past (faded) and future (bright) orbit path
  • A dashed visibility footprint circle showing the area where the satellite is above the horizon (0° elevation)
  • A label with the satellite name and current altitude

Each satellite is drawn in a different colour (cycling through 8 colours) for easy identification when tracking multiple satellites.

TLE Updates

Two-Line Element sets (TLEs) are automatically updated from Celestrak every 24 hours. You can also force an update from the dashboard. Satellite position accuracy degrades without current TLEs — keep your Pi-Clock connected to the internet for best results.

Dashboard: Toggle on/off, adjust opacity. The satellite selection card appears when enabled, allowing you to choose which satellites to track.

Performance: Satellite orbits are propagated on every frame. Tracking many satellites simultaneously increases CPU load. On a Pi Zero W, limit to 3–5 satellites for best performance.

Environmental Data Layers

Lightning Strikes

Updates: 1s Default: Off Requires Network

Real-time lightning strikes from the Blitzortung.org community lightning detection network. Strikes are received via a TLS WebSocket connection and plotted on the map as small lightning bolt icons.

Visual Effect

When a new strike arrives, it appears as a large (3x) bright white lightning bolt for one frame to attract attention, then immediately shrinks to normal size and fades through a colour sequence:

  • White — fresh strike (first second)
  • Yellow — ageing
  • Deep red — about to disappear

After the fade time expires, the strike disappears. The fade time is configurable from the dashboard: 15 seconds (default), 30, 60, or 120 seconds.

Data Source

Blitzortung.org is a community-operated lightning detection network with volunteer stations worldwide. Coverage is densest in Europe and North America. Data arrives in real-time (1–3 second latency from the actual strike).

Network required: The fetcher thread only runs when this layer is enabled. Disabling the layer closes the WebSocket connection and stops all network traffic.

Dashboard: Toggle on/off. When enabled, a settings card appears with a fade time dropdown.

Earthquakes

Updates: 60s (polling) Default: Off Requires Network

Significant earthquakes (magnitude 4.5+) from the USGS Earthquake Hazards Program. The USGS GeoJSON feed is polled every 60 seconds and events are plotted on the map at their epicentre.

Visual Effect

Each earthquake is shown with:

  • A seismograph-style zigzag icon at the epicentre, sized by magnitude
  • Concentric ripple rings radiating outward — tightly packed, bright at the centre, fading toward the edge
  • A magnitude label (e.g., "M5.2") for events M5.0 and above

Events fade out over the final 25% of the display window. The display time is configurable from the dashboard: 30 minutes to 24 hours (default 24 hours, matching the USGS daily feed window).

Data Source

USGS Earthquake Hazards Program (earthquake.usgs.gov). Public domain US federal data, no authentication required. The M4.5+ daily feed typically contains 15–60 events and is 20–80 KB per fetch.

Network required: The fetcher thread only runs when this layer is enabled. Disabling the layer stops polling entirely.

Dashboard: Toggle on/off. When enabled, a settings card appears with a display time dropdown.

Information Layers

Home QTH Marker

Updates: 60s Default: On No Network

Marks your home station location with a crosshair marker (blue inner dot, white outer ring) and your callsign label. The marker only appears when QTH latitude and longitude are both set to non-zero values.

Dashboard: Toggle on/off, adjust opacity. Set your QTH position and callsign in the Station Settings section.

Clock / HUD

Updates: 1s or 60s Default: On No Network

Displays time and date in the top corners of the screen:

  • Top left: UTC time and date
  • Top right: Local time (system timezone) and date, with timezone abbreviation (e.g., "BST", "EDT")

Text is rendered with a shadow for readability on any background. Font sizes scale automatically with the display resolution.

Single-core Pi: On the Pi Zero W and Pi 1, the clock displays hours and minutes only (no seconds) and updates every 60 seconds instead of every second. This saves significant CPU on single-core hardware.

Dashboard: Toggle on/off, adjust opacity.

News Ticker

Updates: 15min (feed refresh) Default: Off Requires Network

A full-width bar at the bottom of the screen showing scrolling news headlines from ham radio and space weather sources. The ticker avoids overlapping with side-panel applets by respecting their margins.

Display Modes

  • Smooth scroll — continuous horizontal scrolling (multi-core Pi only)
  • Headlines — headline flies in (0.5s), title holds for 5 seconds, description slides through in chunks (15s hold), then the next headline. This is the default mode on single-core Pi for CPU efficiency.
