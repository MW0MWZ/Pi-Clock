Configuration Reference

Complete reference for all Pi-Clock configuration files.

Pi-Clock stores configuration in plain-text files on the persistent /data partition. All files use a simple key=value format with # comments. The dashboard reads and writes these files, but you can also edit them directly via SSH.

Live reload: When a configuration file is changed (by the dashboard or manually), the dashboard writes a trigger file at /tmp/pi-clock-reload. The renderer checks for this file on every frame, reloads the configuration, and deletes the trigger. Changes take effect within 1–2 seconds — no reboot required.

Renderer Configuration

File: /data/etc/pi-clock-renderer.conf

The main configuration file for the display renderer. Contains station settings, DX cluster parameters, propagation settings, and display preferences.

# Station settings
CALLSIGN=M1ABC
QTH_LAT=51.5
QTH_LON=-0.1
GRID_SQUARE=IO91wm

# Display settings
CENTER_LON=0.0
DISPLAY_RESOLUTION=native

# DX Cluster settings
DX_CLUSTER_HOST=dxc.ve7cc.net
DX_CLUSTER_PORT=7373
DX_DISTANCE=0
DX_BANDS=0FFF
DX_SPOT_AGE=900

# Propagation settings
ANTENNA_GAIN=0
PROP_BANDS=FF

# System settings (set by pi-clock-config at boot)
HOSTNAME=pi-clock
TIMEZONE=UTC

Key Reference

KeyTypeDefaultDescription
CALLSIGNString(empty)Your callsign, shown on the QTH marker.
QTH_LATFloat0.0Home latitude (-90 to 90). QTH marker hidden if both lat and lon are 0.
QTH_LONFloat0.0Home longitude (-180 to 180).
GRID_SQUAREString(auto)6-character Maidenhead locator. Auto-calculated from lat/lon if blank.
CENTER_LONFloat0.0Map centre longitude. Set to your longitude to centre the map on your location.
DISPLAY_RESOLUTIONStringnativeOne of: native, 720p, 1080p, 1440p, 4k. Requires reboot.
DX_CLUSTER_HOSTStringdxc.ve7cc.netDX cluster server hostname.
DX_CLUSTER_PORTInteger7373DX cluster server port.
DX_DISTANCEInteger0Max spot distance in km (0 = unlimited).
DX_BANDSHex0FFF12-bit band mask (160m=bit0 through 2m=bit11). 0FFF = all bands.
DX_SPOT_AGEInteger900Spot expiry time in seconds. 900 = 15 minutes.
ANTENNA_GAINInteger0Antenna gain in dBi (-12 to 12). Affects propagation predictions.
PROP_BANDSHexFF8-bit band mask for propagation heat map (80m=bit0 through 10m=bit7).
LIGHTNING_FADE_MSInteger15000Lightning strike fade time in milliseconds (5000–120000).
EARTHQUAKE_DISPLAY_SInteger86400Earthquake display time in seconds (60–86400). Default 24 hours.

User Configuration (Persistent)

File: /data/etc/pi-clock-user.conf

Stores settings that must survive OS upgrades, including the user password hash, hostname, timezone, and SSH configuration. This file is written by pi-clock-config at boot and by the dashboard when system settings change.

Do not edit the password hash manually unless you know what you are doing. Use the dashboard or pi-clock-config.txt to set passwords. The hash format is SHA-512 from /etc/shadow.

Slot Configuration (Boot Partition)

File: /boot/firmware/slot.conf

Controls the A/B slot upgrade system. Written atomically by the upgrade and rollback scripts. Do not edit manually unless troubleshooting a boot failure.

ACTIVE_SLOT=A
PREVIOUS_SLOT=B
PENDING=false
BOOT_TRIES=0
SLOT_A_VERSION="Pi-Clock OS v1.0.0 (2026-04-10)"
SLOT_B_VERSION="Pi-Clock OS v0.9.0 (2026-04-01)"
KeyDescription
ACTIVE_SLOTA or B — the slot that will be booted.
PREVIOUS_SLOTA or B — the fallback slot for rollback.
PENDINGtrue or false — whether a new slot is being validated.
BOOT_TRIESInteger — incremented on each boot when PENDING=true. Rollback after 3.
SLOT_A_VERSIONVersion string for Slot A (or empty if not installed).
SLOT_B_VERSIONVersion string for Slot B (or empty if not installed).
