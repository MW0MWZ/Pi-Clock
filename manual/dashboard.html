Web Dashboard

Configure and manage Pi-Clock from any browser on your network.

Accessing the Dashboard

The dashboard runs on your Pi-Clock and is accessible over HTTPS from any device on the same network:

https://pi-clock.local

If mDNS/Bonjour is not available on your network, use the Pi's IP address instead (visible in the system info applet on the display, or find it via your router's DHCP lease table).

Self-signed certificate: The dashboard uses an auto-generated ECDSA P-256 certificate. Your browser will show a security warning on first visit — this is expected. Accept the certificate to proceed. The certificate is valid for 10 years and persists across OS upgrades.

Login

The dashboard requires authentication. Default credentials:

  • Username: pi-clock
  • Password: pi-clock (change this immediately!)

Sessions expire after 30 minutes of inactivity. Login attempts are rate-limited to 5 per minute per IP address to prevent brute-force attacks.

Station Settings

Set your amateur radio station details. These are used by the QTH marker, DX cluster distance filter, MUF predictions, and VOACAP propagation model.

SettingDescription
CallsignYour amateur radio callsign, displayed on the QTH marker.
QTH LatitudeYour station latitude (-90 to +90 degrees).
QTH LongitudeYour station longitude (-180 to +180 degrees).
Grid SquareMaidenhead locator (6-character, e.g., IO91wm). Auto-calculated from lat/lon if left blank.

Click the "Use My Location" button to automatically fill in your latitude, longitude, and grid square using your browser's geolocation API. This requires HTTPS and your browser's permission.

Display Settings

SettingDescription
Center LongitudeThe longitude at the horizontal centre of the map (-180 to +180). Set this to your local longitude to centre the map on your part of the world. Default: 0 (Greenwich).
Display ResolutionChoose from Native (auto-detect from monitor), 720p, 1080p, 1440p, or 4K. Changes take effect after reboot. See the Hardware Guide for per-model limits.

Layer Controls

The layer controls section lets you toggle each of the 15 display layers on or off, and adjust their opacity from 0% (fully transparent) to 100% (fully opaque). See Display Layers for a full description of each layer.

Two layers are always on and cannot be disabled: Base Map and Daylight. These form the foundation of the display.

When you enable certain layers, additional configuration cards will appear on the dashboard:

  • Enable DX Cluster Spots → DX Cluster Settings card appears
  • Enable Satellite Tracking → Satellite Selection card appears
  • Enable Propagation Heat Map → Propagation Settings card appears
  • Enable Lightning Strikes → Lightning Settings card appears
  • Enable Earthquakes → Earthquake Settings card appears
  • Enable News Ticker → Ticker Sources card appears
  • Rain Radar, Cloud Cover, Wind, and Aurora are simple toggles with opacity control — no additional configuration needed

Changes are applied immediately to the display — no reboot required.

DX Cluster Settings

This card appears when the DX Cluster Spots layer is enabled. It controls how DX spots are filtered and displayed.

Distance Filter

Limits displayed spots to those within a certain distance of your QTH. Great-circle distance is calculated from your home coordinates. Options:

  • All spots (no filter)
  • 500 miles / 800 km
  • 1,000 miles / 1,600 km
  • 2,000 miles / 3,200 km
  • 3,000 miles / 4,800 km
  • 5,000 miles / 8,000 km

Spot Age

How long spots remain on the map before expiring. Older spots fade in opacity as they approach expiry. Options: 5, 10, 15 (default), 30, or 60 minutes.

Band Filter

Toggle individual bands on or off. Each band has a colour-coded button matching the band colours used on the map. You can enable any combination of the 12 bands from 160m through 2m.

Satellite Selection

This card appears when the Satellite Tracking layer is enabled. It shows a list of available amateur radio satellites grouped by category (Popular, FM Repeaters, Linear Transponders, Digipeaters, Weather). Toggle each satellite on or off individually.

Performance: Each tracked satellite requires real-time SGP4 orbit propagation on every frame. On a Pi Zero W, limit to 3–5 satellites. On a Pi 4/5, you can comfortably track all available satellites simultaneously.

Lightning Settings

This card appears when the Lightning Strikes layer is enabled. Data is sourced from the Blitzortung.org community network via a real-time WebSocket connection.

SettingDescription
Fade TimeHow long each strike remains visible on the map. Options: 15 seconds (default), 30 seconds, 60 seconds, or 120 seconds. Longer times show more strike history but can clutter the map during active storms.

Earthquake Settings

This card appears when the Earthquakes layer is enabled. Data is sourced from the USGS Earthquake Hazards Program (M4.5+ globally).

SettingDescription
Display TimeHow long each earthquake remains visible on the map. Options: 30 minutes, 1 hour, 2 hours, 4 hours, 12 hours, or 24 hours (default). The USGS feed provides events from the past 24 hours, so the default shows all available data.

Propagation Settings

This card appears when the Propagation Heat Map layer is enabled.

SettingDescription
Band SelectionChoose which HF bands (80m through 10m) are included in the propagation cycle. Each band has a colour-coded toggle button.
Antenna GainYour antenna gain in dBi (-12 to +12). Affects the predicted signal reliability — higher gain means the model predicts more paths as "open". Default: 0 dBi.

Applet Controls

Enable or disable each of the six applet panels and choose whether they appear on the left or right side of the display. See Applets for full details on each applet.

Ticker Sources

This card appears when the News Ticker layer is enabled. Configure:

SettingDescription
Display ModeSmooth scroll (continuous horizontal motion, multi-core only) or Headlines (fly-in, hold, next — works on all hardware). Single-core Pi is forced to Headlines mode regardless of this setting.
News SourcesToggle each source independently: NOAA Alerts, DX World, ARRL News, RSGB GB2RS, Southgate ARC.

System Management

The System section of the dashboard provides tools for managing the Pi-Clock hardware and OS.

System Information

Displays the Pi model, hostname, uptime, CPU architecture and cores, display resolution, refresh rate, and OS version. Also shows the A/B slot status (active slot, versions, and whether an upgrade is pending).

WiFi Configuration

SettingDescription
SSIDWiFi network name (max 32 characters).
PasswordWPA2 passphrase (8–63 characters). Stored as a hashed PSK, never in plain text.
Country Code2-letter ISO country code for WiFi regulatory domain (e.g., GB, US, DE). Defaults to GB.

WiFi changes are applied immediately via wpa_cli reconfigure — no reboot needed.

Hostname

Change the system hostname (letters, digits, and hyphens; max 63 characters). The mDNS name updates to {hostname}.local. Applied immediately and persisted across OS upgrades.

Password

Change the dashboard login password. Requires the current password for verification. Must be at least 8 characters. All active sessions are invalidated when the password is changed.

Reboot & Shutdown

Reboot or shut down the Pi-Clock from the dashboard. Both actions show a confirmation dialog before proceeding.

Package Upgrade

Updates all Alpine Linux packages (including the Pi-Clock renderer, dashboard, and maps) to the latest versions available in the configured repositories. Equivalent to apk update && apk upgrade.

OS Image Upgrade

Downloads and installs a complete new OS image to the inactive A/B slot. The dashboard shows real-time progress via Server-Sent Events as the upgrade progresses through seven stages: manifest check, download, verify, format, extract, boot update, and finalise.

After the upgrade completes, reboot to activate the new slot. The boot watchdog will automatically roll back if the new slot fails to boot successfully three times. See Installation & OS for full details on the A/B upgrade system.

Update Check

Checks for a newer OS version without installing it. Shows the installed version and the latest available version.

Rollback

Manually switch back to the previous OS slot. Useful if an upgrade introduced unexpected behaviour that didn't trigger the automatic three-strike rollback.

Security

The dashboard implements several security measures appropriate for a home network appliance:

  • HTTPS only — HTTP (port 80) automatically redirects to HTTPS (port 443)
  • HSTS — Strict-Transport-Security header forces HTTPS for 1 year
  • Secure cookies — HttpOnly, Secure, SameSite=Strict session cookies
  • Rate limiting — 5 login attempts per minute per IP address
  • Constant-time password comparison — prevents timing attacks
  • Content Security Policy — restricts scripts and styles to self-origin only
  • Clickjacking protection — X-Frame-Options: DENY
  • Request body limit — 1 MB maximum to prevent memory exhaustion on Pi Zero W
  • Input validation — all user input is validated server-side with regex whitelists

mDNS / Bonjour Discovery

Pi-Clock advertises itself on the local network via mDNS (multicast DNS / Bonjour). This allows you to access the dashboard at https://pi-clock.local (or https://{hostname}.local if you changed the hostname) without needing to know the IP address.

mDNS works automatically on macOS, iOS, and most Linux distributions. On Windows, you may need to install Bonjour Print Services or use the IP address directly.
