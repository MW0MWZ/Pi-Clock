Applets

Side-panel information displays that overlay the world map.

Applets are semi-transparent information panels that stack along the left or right edges of the screen. They provide detailed data at a glance without obscuring the entire map. Each applet can be independently enabled/disabled and positioned on either side of the display.

When multiple applets are enabled on the same side, they stack from the bottom of the screen upward. All applets on the same side share the same width (determined by the widest applet on that side).

DX Cluster Feed

Key: dxfeed Updates: 30–60s Requires Network ~250px wide

A compact scrolling feed of recent DX cluster spots. Displays up to 12 lines showing callsign, frequency, spotter, and any comments. Uses a monospace font for data density.

Lines are brightness-graded: the newest spot is brightest, older spots gradually fade. This makes it easy to spot fresh activity at a glance.

The feed is populated from the same DX cluster telnet connection used by the DX Spots map layer. If the DX Spots layer is not enabled, the feed applet will still work — it shares the background data connection.

Dashboard: Enable/disable, choose left or right side.

Solar Weather

Key: solar Updates: 15min Requires Network ~300px wide

Displays live solar indices from NOAA's Space Weather Prediction Center. This is the single most important panel for HF operators — it tells you at a glance whether conditions are good, marginal, or poor.

Indices Displayed

Kp Planetary K-index — geomagnetic disturbance (lower is better) ≤3 ≤5 >5
Bz Interplanetary magnetic field z-component (positive is better) ≥0 nT ≥-5 nT <-5 nT

SDO Sun Images

Below the indices, the applet displays a cycling set of SDO (Solar Dynamics Observatory) sun images in five different wavelengths, changing every 60 seconds. These show coronal holes, active regions, and flare activity.

Data source: NOAA SWPC JSON API (services.swpc.noaa.gov). Updated every 15 minutes.

Dashboard: Enable/disable, choose left or right side.

MUF Estimate

Key: muf Updates: 1min Works Offline ~250px wide

Estimates the Maximum Usable Frequency (MUF) from your QTH in five directions:

  • N — North
  • E — East
  • S — South
  • W — West
  • NVIS — Near Vertical Incidence Skywave (local/regional coverage)

How It Works

The MUF prediction is computed entirely offline using:

  • Real solar position via the Meeus algorithm
  • F2-layer critical frequency model based on latitude and sunspot number
  • Solar zenith angle at the path midpoint for each direction
  • D-layer absorption estimates (ITU-R P.533)

When connected to the internet, the model uses the live Solar Flux Index from NOAA for better accuracy. When offline, it defaults to SFI=100.

Requires: QTH latitude and longitude to be set.

Dashboard: Enable/disable, choose left or right side.

VOACAP Prediction

Key: voacap Updates: 1min (recalc) / 10s (cycle) Uses NOAA Data ~320px wide

A band-by-hour heat map showing predicted HF propagation for 8 amateur bands (80m through 10m) across all 24 UTC hours. This is the most detailed propagation display available in Pi-Clock.

How to Read It

The heat map is organised as a grid with bands on one axis and UTC hours on the other. Each cell uses opacity to indicate propagation likelihood: more opaque (solid) means higher probability of a usable path, more transparent means propagation is less likely. Fully transparent cells have no predicted propagation for that band/hour/distance combination.

Distance Brackets

The applet cycles through 8 distance brackets every 10 seconds, from NVIS (local) through to antipodal (maximum distance). This lets you see propagation predictions for different path lengths.

Model Details

The VOACAP-style model includes:

  • F2 layer critical frequency (latitude/sunspot number model)
  • D-layer absorption (time-of-day and latitude dependent)
  • Multi-hop obliquity factors for different distances
  • Kp-index degradation for geomagnetic storms

Requires: QTH latitude and longitude. Uses live SFI and Kp from NOAA when available.

Dashboard: Enable/disable, choose left or right side. Antenna gain and band selection configurable.

System Info

Key: sysinfo Updates: 10–30s No Network ~250px wide

Displays hardware and software status information:

  • Hostname — the system hostname (and network address)
  • IP Address — IPv4 (and IPv6 if available)
  • OS — operating system name and version
  • CPU — architecture and core count
  • Display — current resolution and refresh rate
  • Version — renderer version number
  • Uptime — system uptime (refreshed on each render)

This applet is particularly useful for confirming the IP address of your Pi-Clock when you need to access the dashboard, and for verifying the display resolution detected at boot.

Dashboard: Enable/disable, choose left or right side.

Active Features

Key: features Updates: on config change No Network

Shows a summary of which layers and applets are currently enabled. Useful as a quick visual reference to confirm your configuration without opening the dashboard.

Dashboard: Enable/disable, choose left or right side.

Applet Configuration

Applets are configured via the dashboard or by editing /data/etc/pi-clock-applets.conf:

# Applet configuration — format: name=enabled,side
dxfeed=1,right
solar=1,left
muf=0,left
voacap=1,right
sysinfo=1,left
features=1,right

Set the first value to 1 (enabled) or 0 (disabled). The second value is the screen side: left or right.

Layout tip: Balance your applets between left and right sides to keep the central map area clear. Propagation-heavy setups work well with solar and MUF on the left, VOACAP and DX feed on the right.
